Markdown Calendar

A native macOS calendar that reads a folder of markdown files. Each file's date: front-matter field decides where it shows up. Built for teachers who want their lesson notes, requirements lists, and weekly planning all in one place — without giving up plain-text portability.

↓ Download for macOS

macOS 13 Ventura or later · ~10 MB · MIT licensed

Or build from source — see the GitHub repo

What it does

Point it at any folder of .md files where each file has a date in its front matter and you get three views over them: a month grid, a Mon–Fri week with hourly time slots, and a chronological timeline. A right-hand panel shows everything on the day you've clicked, with full markdown rendering. A toggleable left-hand panel scrapes ## Requirements sections from the week's notes and gives you an aggregated weekly checklist — handy for working out how many photocopies and whiteboard markers you need before the week starts.

The menu-bar tray shows today's entries at a glance — click any one to open it. Live folder watching means changes you make in another editor (Obsidian, VSCode, plain TextEdit) appear instantly without a refresh.

Example file

---
title: Year 9 Maths — quadratics
date: 2026-05-12 09:30
end: 2026-05-12 10:30
tags: [year-9, maths]
---

Today's lesson covers quadratic equations.

## Requirements

- whiteboard markers
- worksheets (printed × 28)
- graphing calculators

Tags get assigned colors automatically by hashing the tag name through a 24-slot palette, so year-7 and year-9 always come out as visibly different colors no matter which Mac you're on.

Install

Download the .dmg above. Double-click to mount, drag Markdown Calendar onto the Applications folder, then eject the disk image.

First launch: macOS will warn that "Markdown Calendar cannot be opened because Apple cannot check it for malicious software." This is normal for apps that aren't notarized through Apple's $99/year Developer Program. To bypass it, right-click Markdown Calendar in the Applications folder and choose Open, then click Open in the dialog that appears. macOS remembers and won't ask again.

On launch you'll see a welcome screen — click Choose folder… and pick a folder of markdown files. The app remembers the folder between launches.

Front matter reference

FieldTypeNotes
daterequiredYYYY-MM-DD or YYYY-MM-DD HH:MM. Falls back to created or published.
titleoptionalIf absent, the filename is used.
endoptionalEnd time. Also accepts end_date or endDate.
durationoptionalLike 30m, 1h 15m, or just minutes.
tagsoptionalYAML array [a, b] or comma-separated string.

Source code

Pure Swift + SwiftUI + AppKit. No webview, no Electron. About 2,800 lines, MIT licensed. The repository is at github.com/YOUR-USERNAME/markdown-calendar — issues and pull requests welcome.

The build pipeline is a single shell script: ./make-app.sh wraps the swift-built binary into a .app bundle, ./make-dmg.sh --pretty turns that into a .dmg installer. No Xcode project required, only the Swift toolchain that ships with macOS Command Line Tools.
